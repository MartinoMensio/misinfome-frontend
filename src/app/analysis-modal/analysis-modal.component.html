Learn more about the analysis

MisinfoMe starts by retrieving the public tweets created or retweeted by the Twitter account provided in the search bar. Then it goes through these tweets to extract any external links and look for them in a database of fact-checkings.

Our database consists of over 120 thousand ClaimReview annotations published by Fact-Checkers. ClaimReview is a standard schema used for fact-checking reviews of claims, and from it we collect the URLs (if present) of the fact-checked article, and the validation outcome. MisinfoMe database also contains published source reliability assessments.

Initially, the last 500 tweets published by the given account are analysed. The search can be expanded to include all the tweets available from the given account, but this process may take a few minutes to complete!

Credibility scores

MisinfoMe assigns a credibility score, ranging between -1 (not credible) to 1 (credible), for each analysed tweet.

The score calculation considers all the fact-checking information available for the link in the tweet, including fact-checking assessments and the reputation of the source(s) of information as assessed by fact-checkers. If assessments exist from multiple fact-checkers then they will be combined using a set of rules. More detail can be found in the following paper:

Mensio, Martino and Alani, Harith (2019). News Source Credibility in the Eyes of Different Assessors. In: Conference for Truth and Trust Online, 4-5 Oct 2019, London, UK

Privacy

MisinfoMe uses the official Twitter application programming interface (API) to collect the data it needs for the analyses. Only public accounts and tweets are analysed. MisinfoMe has no access to any blocked accounts and private messages.

No information about the users of MisinfoMe is requested, collected, or stored.
